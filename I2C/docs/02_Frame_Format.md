# I2C Frame Format

A typical I2C transaction consists of several fields exchanged between the Master and Slave devices.

The Master initiates communication, selects the target slave, specifies the transfer direction, exchanges data, and terminates the transaction.

---

# Typical I2C Transaction

A complete I2C transaction follows the sequence shown below.

```text
┌───────┬─────────────┬─────┬──────────┬─────────┬──────────┬─────────┬──────────┬──────┐
│ START │ 7/10 Address│ R/W │ ACK/NACK│  DATA   │ ACK/NACK │  DATA   │ ACK/NACK │ STOP │
└───────┴─────────────┴─────┴──────────┴─────────┴──────────┴─────────┴──────────┴──────┘
```

---

# Frame Components

Each field in the I2C frame has a specific purpose.

---

## START

Generated by the Master.

Purpose:

* Initiate communication
* Wake-up all slave devices
* Acquire ownership of the bus

Only during START condition can SDA transition while SCL remains HIGH.

---

## Slave Address

The slave address uniquely identifies the destination device.

Supported formats:

* 7-bit Address
* 10-bit Address

---

### 7-bit Address

Most commonly used.

Example:

```text
AHT20 Address

0x38


Binary

00111000
```

Total addresses:

```text
128
```

Address range:

```text
0x00 – 0x7F
```

---

### 10-bit Address

Provides additional addressing capability.

Address range:

```text
0x000 – 0x3FF
```

Total addresses:

```text
1024
```

---

# R/W Bit

The Read/Write bit immediately follows the slave address.

It specifies the communication direction.

---

## Write Operation

Bit value:

```text
0
```

Meaning:

Master transmits data.

Slave receives data.

---

Example

```text
Address + Write


0x38 + 0
```

---

## Read Operation

Bit value:

```text
1
```

Meaning:

Master requests data.

Slave transmits data.

---

Example

```text
Address + Read


0x38 + 1
```

---

# ACK/NACK

Each transmitted byte is followed by an acknowledgment bit.

This mechanism provides error detection.

---

## ACK

Bit value

```text
0
```

Meaning

Receiver successfully received data.

Examples:

Slave accepted address.

Slave accepted data.

Slave completed command.

---

## NACK

Bit value

```text
1
```

Meaning

Receiver rejected data.

Possible causes:

* Invalid slave address
* Device not connected
* Sensor busy
* End of read operation
* Bus error

---

# DATA Field

Data is exchanged in units of 8 bits.

Each byte requires an acknowledgment.

---

## Example Data Bytes

```

```

Reserved byte

```text
0x00
```

# STOP

STOP terminates communication.

The Master releases bus ownership.

---

State

```text
SDA : LOW → HIGH

SCL : HIGH
```

---

Purpose

* End transaction
* Release bus
* Return to Idle state

# Important Notes

### Data Width

I2C transfers data in 8-bit units.

---

### ACK Responsibility

Transmitter sends data.

Receiver generates ACK.

---

### Bus Ownership

Only one Master can actively communicate.

Arbitration resolves conflicts.

---

### Addressing

Multiple devices can coexist on the same bus.

Unique addresses are mandatory.

---

# Summary

The I2C frame format defines how devices exchange information over the shared bus. Understanding the frame structure is essential for debugging transactions, validating communication using Linux utilities, and developing reliable embedded applications.
