# Overview

## What is I2C?

**I2C (Inter-Integrated Circuit)** is a **2-wire, serial, synchronous communication protocol** developed by Philips Semiconductors (now NXP) for communication between integrated circuits.

It is one of the most widely used communication protocols in embedded systems due to its simplicity, low pin count, and support for multiple devices on a shared bus.

I2C is commonly used to interface:

* Temperature Sensors
* Humidity Sensors
* Pressure Sensors
* EEPROMs
* RTC Devices
* OLED Displays
* GPIO Expanders
* ADC/DAC Converters
* PMICs

---

# I2C Characteristics

| Property                    | Description    |
| --------------------------- | -------------- |
| Communication Type          | Serial         |
| Synchronization             | Synchronous    |
| Number of Wires             | 2              |
| Communication Direction     | Bi-Directional |
| Multi-Master Support        | Yes            |
| Multi-Slave Support         | Yes            |
| Bus Type                    | Shared Bus     |
| Address Based Communication | Yes            |

---

# I2C Bus Lines

I2C communication uses only two signals.

## SDA – Serial Data Line

The SDA line is used to transfer data between Master and Slave devices.

Characteristics:

* Bi-directional
* Open-drain/Open-collector
* Shared among all devices
* Requires pull-up resistor

---

## SCL – Serial Clock Line

The SCL line provides synchronization between devices.

Characteristics:

* Generated by Master
* Open-drain
* Shared bus
* Supports Clock Stretching

# Supported Topologies

I2C supports multiple communication architectures.

---

## Single Master Single Slave

```text
Master
   │
   │
Slave
```

Suitable for simple embedded systems.

Examples:

* MCU + EEPROM
* MCU + RTC

---

## Single Master Multiple Slaves

```text
                  Slave1
                     │
Master ──────────────┼──── Slave2
                     │
                  Slave3
```

Each slave device has a unique address.

Examples:

* AHT20
* DS3231
* SSD1306

---

## Multi-Master Multi-Slave

```text
Master1 ─────┐
             │
             ├──────── Shared I2C Bus ───── Slave1
             │
Master2 ─────┘                         ├──── Slave2
                                       └──── Slave3
```

Only one Master can actively communicate at a given time.

Arbitration is used to determine bus ownership.

---

# Addressing Scheme

Slave devices are selected using an address.

Two addressing modes are supported.

---

## 7-bit Addressing

Most commonly used.

Address range:

```text
0x00 – 0x7F
```

Maximum devices:

```text
128
```

Examples:

```text
AHT20    -> 0x38

DS3231   -> 0x68

SSD1306  -> 0x3C
```

---

## 10-bit Addressing

Used when additional slave addresses are required.

Address range:

```text
0x000 – 0x3FF
```

Maximum devices:

```text
1024
```

# Advantages

* Only two wires required
* Low PCB routing complexity
* Supports multiple devices
* Low pin count
* Well supported by Linux
* Widely available peripherals

---

# Limitations

* Slower than SPI
* Requires pull-up resistors
* Bus capacitance limitations
* Additional protocol overhead

---
