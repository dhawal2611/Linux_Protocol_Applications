# Start, Stop and Repeated Start Conditions

I2C communication is controlled using special bus conditions generated by the Master device.

These conditions indicate the beginning and end of a transaction and allow multiple transactions to be chained together without releasing the bus.

The three important bus conditions are:

* START
* STOP
* Repeated START

---

# Bus Idle State

Before communication begins, the bus remains in an idle state.

Bus Idle State

```text id="g7pj6e"
SCL = HIGH

SDA = HIGH
```

All slave devices continuously monitor the bus while it remains idle.

---

# START Condition

A START condition initiates communication between the Master and Slave devices.

It acts as a wake-up call to all devices connected to the bus.

---

## Purpose

The START condition is used to:

* Acquire bus ownership
* Begin a transaction
* Notify slaves that communication is starting
* Synchronize all devices on the bus

---

## State Transition

The Master generates a START condition by changing the SDA line while SCL remains HIGH.

```text id="w0w2v7"
SDA : HIGH → LOW

SCL : HIGH
```

This is one of the only instances in the I2C protocol where SDA is permitted to change state while SCL is HIGH.

---

## Timing Representation

```text id="0xrmw9"
SCL  ──────────────────────────────

SDA  ─────────────┐
                  │
                  └────────────────
```

---

## Bus Activity After START

Immediately following START, the Master transmits:

* Slave Address
* R/W Bit

The addressed slave responds with an ACK.

---

## Example

AHT20 measurement transaction.

```text id="t3mx6v"
START

0x38 + Write

ACK
```

---

# STOP Condition

A STOP condition indicates the completion of communication.

It informs all slave devices that the bus is available again.

---

## Purpose

STOP is used to:

* Terminate communication
* Release bus ownership
* Return bus to idle state

---

## State Transition

The Master generates a STOP condition by changing SDA while SCL remains HIGH.

```text id="hjxy1h"
SDA : LOW → HIGH

SCL : HIGH
```

---

## Timing Representation

```text id="zyynmv"
SCL  ──────────────────────────────

SDA  ______________________
                            │
                            └──────────────
```

---

## Bus State After STOP

After STOP:

```text id="cqlc1k"
SDA = HIGH

SCL = HIGH
```

The bus is considered idle.

Any Master may initiate a new transaction.

---

## Example

```text id="1ps0kh"
Data Byte


ACK


STOP
```

---

# Repeated START Condition

A Repeated START allows a Master to initiate a new transaction without first issuing a STOP condition.

This enables the Master to retain ownership of the bus.

---

## Why Repeated START is Needed

In multi-master systems, issuing a STOP condition releases the bus.

Another Master could seize the bus before the original Master completes its operation.

Repeated START avoids this issue.

---

## Applications

Repeated START is commonly used for:

* Sensor register reads
* EEPROM reads
* RTC reads
* Combined write-read transactions

---

## State Transition

The Master generates another START before sending STOP.

The bus is not released.

---

## Timing Representation

```text id="7i2z4z"
START

Address + Write

ACK


Register Address

ACK


Repeated START


Address + Read

ACK


Read Data


STOP
```

---

# Example Using AHT20

The AHT20 requires a measurement trigger before data can be read.

The communication sequence is:

```text id="dfvfx9"
START


0x38 + Write


ACK


0xAC


ACK


0x33


ACK


0x00


ACK



Repeated START


0x38 + Read


ACK



Read Byte 1


ACK


Read Byte 2


ACK


Read Byte 3


ACK


Read Byte 4


ACK


Read Byte 5


ACK


Read Byte 6


ACK


Read Byte 7

```

# Comparison of Bus Conditions

| Condition      | SDA Transition | SCL State | Purpose                                     |
| -------------- | -------------- | --------- | ------------------------------------------- |
| START          | HIGH → LOW     | HIGH      | Begin Communication                         |
| STOP           | LOW → HIGH     | HIGH      | End Communication                           |
| Repeated START | HIGH → LOW     | HIGH      | Begin New Transaction Without Releasing Bus |

---

# Important Notes

### START and STOP are Unique

These are the only conditions where SDA is allowed to change while SCL remains HIGH.

During normal data transfer:

```text id="v7r7b7"
SDA changes only when SCL is LOW
```

---

### Repeated START Preserves Bus Ownership

Repeated START is particularly useful in systems containing multiple Masters.

It guarantees that another Master cannot interrupt an ongoing combined transaction.

---

# Summary

START, STOP, and Repeated START conditions control the flow of communication in the I2C protocol.

Understanding these conditions is essential when debugging transactions using logic analyzers, validating communication with Linux I2C utilities, or developing embedded applications that interact with sensors, memories, and peripheral devices.
